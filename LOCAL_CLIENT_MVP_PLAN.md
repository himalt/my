# 上货助手本地客户端 MVP 执行标准

## 目标

把现有网页系统改造成：

```text
无登录本地客户端
+ 本地采集
+ 本地图片处理
+ 本地商品处理
+ 本地上货
+ 本地数据清理
```

核心闭环必须是：

```text
采集商品 → 下载/处理图片 → 商品处理 → 上货 RPA → 成功/失败回写 → 失败重试 → 导出/清理
```

第一版不做登录、计费、CDN、对象存储、云端商品库、多用户协作。云端授权和计费后续再接入。

## 总原则

- 先不做登录：第一版不接云端授权、不接支付、不接 CDN。
- 先不改大架构：复用现有网页、FastAPI、SQLite、前端页面。
- 先跑通闭环：采集、图片、处理、上货、结果回写必须连起来。
- 先本地化：所有商品数据、图片、日志、导出文件都存在用户电脑。
- 先稳定再美化：页面样式不是重点，批量任务稳定性才是重点。
- 先小批量再放大：先跑通 1 个商品，再 20 个，再 100 个，再 1000 个。
- 不要重写整个系统；只做支撑本地客户端 MVP 的必要改造。

## 阶段一：本地运行基础

### 目标

用户双击程序后，可以在本机打开完整系统。

### 执行步骤

1. 确认当前项目启动方式：后端启动入口、前端静态文件路径、SQLite 数据库路径、图片保存路径。
2. 统一本地数据目录：开发期可用 `D:\projects\upload-assistant\data`，打包后使用用户本地应用数据目录。
3. 建立目录结构：

```text
data/
  app.db
  images/
    original/
    preview/
    thumb/
  exports/
  logs/
  temp/
```

4. 所有路径改为相对本地数据目录：数据库、图片、日志、导出文件不能写死开发路径。

### 验收标准

```text
断开业务服务器后，本地页面仍然能打开。
本地能读写 SQLite。
本地能保存图片。
本地能生成日志。
```

## 阶段二：商品采集入库

### 目标

采集结果直接写入本地 SQLite。

### 执行步骤

1. 设计或确认商品表至少包含：

```text
id
source
source_url
title
price
sku_data
attributes
image_urls
local_image_paths
status
created_at
updated_at
error_message
```

2. 设计或确认采集任务表至少包含：

```text
id
task_type
input_value
status
total_count
success_count
failed_count
started_at
finished_at
error_message
```

3. 采集任务必须异步执行：页面创建任务后立即返回，后台采集，页面轮询进度。
4. 采集结果必须去重：按 `source_url` 或平台商品 ID 去重，避免重复商品无限新增。

### 验收标准

```text
输入采集源后，可以生成采集任务。
任务进度可见。
采集成功商品进入本地商品库。
失败商品有失败原因。
重复商品不会无限新增。
```

## 阶段三：图片本地化处理

### 目标

商品图片不再依赖外链，全部下载到本地并生成缩略图。

### 执行步骤

1. 采集完成后自动进入图片下载任务。
2. 图片下载失败要记录原因，并支持重试失败图片。
3. 图片保存规则：

```text
data/images/original/{product_id}_{index}.jpg
data/images/preview/{product_id}_{index}.webp
data/images/thumb/{product_id}_{index}.webp
```

4. 生成三类图片：`original` 用于上货，`preview` 用于详情预览，`thumb` 用于商品列表。
5. 商品列表只使用缩略图，禁止商品列表直接加载原图。
6. 前端图片加懒加载。
7. 没有可用图片的商品进入异常状态，不允许直接上货。

### 验收标准

```text
采集后图片能下载到本地。
商品列表加载的是 thumb。
详情页可以看 preview。
上货使用 original。
断开外链后，已下载图片仍然能显示。
```

## 阶段四：商品处理工作台

### 目标

用户可以在本地页面批量处理商品，并把商品推到待上货状态。

### 商品状态流

固定使用或兼容以下状态：

```text
new              新采集
image_pending    图片待处理
image_ready      图片已完成
needs_review     待处理
ready_upload     待上货
uploading        上货中
uploaded         已上货
upload_failed    上货失败
skipped          已跳过
```

### 执行步骤

1. 商品列表必须分页，每页默认 `20` 或 `50`，禁止一次加载全部商品。
2. 商品列表必须筛选，至少支持：全部、待处理、待上货、上货失败、已上货、缺图。
3. 商品编辑至少支持：标题、价格、SKU、图片、模板/类目相关字段、备注。
4. 批量操作至少支持：批量设为待上货、批量跳过、批量删除、批量重试图片、批量重试上货。

### 验收标准

```text
1000 个商品列表不卡。
每页只显示一小批。
商品可以编辑。
可以批量标记待上货。
状态流转清楚。
```

## 阶段五：上货 RPA 闭环

### 目标

从 `ready_upload` 队列取商品，自动上货，并回写结果。

### 执行步骤

1. 设计或确认上货任务表至少包含：

```text
id
product_id
status
current_step
started_at
finished_at
error_message
screenshot_path
retry_count
```

2. 用户选择商品并点击“开始上货”后，系统创建上货任务，后台逐个执行。
3. RPA 使用本地数据：商品字段来自 SQLite，图片路径来自本地 `original`，店铺/模板配置来自本地设置页。
4. 每一步写日志，例如：打开上货页面、选择店铺、选择模板、填写标题、上传图片、填写 SKU、提交、确认成功。
5. 失败要可复盘：记录失败步骤、错误原因、截图，商品状态改为 `upload_failed`，支持重试。
6. 成功要回写：商品状态改为 `uploaded`，记录上货时间，记录平台返回信息（如果有）。

### 验收标准

```text
单个商品可以从采集跑到上货成功。
20 个商品可以批量上货。
失败商品能看到失败原因。
失败商品能重试。
上货成功后状态正确回写。
```

## 阶段六：本地清理和导出

### 目标

数据不长期保存，但用户可以导出和清理。

### 执行步骤

1. 支持导出 Excel 或 CSV，可按状态导出，可导出上货失败列表和已上货列表。
2. 支持清理已上货商品、7 天前图片、失败任务日志、临时文件。
3. 默认保留最近 7 天数据，已上货成功图片可清理，失败商品不自动清理。

### 验收标准

```text
用户可以导出结果。
上货完成的数据可以清理。
清理后页面不报错。
本地磁盘不会无限增长。
```

## 阶段七：打包成本地客户端

### 目标

用户无需配置环境，双击即可使用。

### 执行步骤

1. 使用 PyInstaller 或类似工具打包后端，包含 FastAPI 后端、静态前端、必要依赖。
2. 启动器双击后自动执行：检查本地数据目录、初始化数据库、启动本地服务、自动打开浏览器。
3. 默认使用 `127.0.0.1:8000`；如果端口占用，自动换端口或提示。
4. 重复双击时不启动多个后端，可直接打开已运行页面。
5. 所有异常写入 `data/logs/app.log`。

### 验收标准

```text
新电脑解压后可运行。
不用安装 Python。
不用手动启动后端。
双击自动打开页面。
采集、图片、处理、上货都能正常跑。
```

## AI 自主执行规则

后续 AI 执行本计划时必须遵守：

```text
你需要自主完成本地客户端 MVP。
不要每完成一个小步骤就问用户。
除非遇到账号、密码、平台验证码、真实上货确认、外部 API 密钥、支付配置，否则继续推进。
优先复用现有代码，不要重写系统。
优先保证采集到上货闭环，不要优先美化页面。
每完成一个阶段，自己运行最小验证。
如果发现 bug，直接修复并重新验证。
最后只汇报结果、改动文件、验证情况、剩余风险。
```

## AI 每次执行固定检查清单

```text
1. 读取项目结构
2. 找到启动入口
3. 找到数据库配置
4. 找到图片路径配置
5. 找到商品处理页接口
6. 找到上货 RPA 入口
7. 制定本轮计划
8. 修改代码
9. 运行最小验证
10. 记录剩余问题
```

## 每轮开发边界

每轮只聚焦一个阶段目标，避免一次性改太多导致失控。

示例：

```text
本轮只做：图片本地化和缩略图。
不要改登录。
不要改支付。
不要改 UI 大布局。
不要改无关功能。
完成后运行验证。
```

## 最终 MVP 验收标准

```text
1. 双击本地启动
2. 能采集商品
3. 能下载图片到本地
4. 能生成缩略图
5. 商品处理页不卡
6. 能编辑商品
7. 能标记待上货
8. 能启动上货
9. 能回写成功/失败
10. 失败能重试
11. 能导出结果
12. 能清理本地数据
```
